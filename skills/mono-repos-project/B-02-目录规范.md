# B-02 目录规范（联邦项目）

联邦项目下根目录为 pnpm workspace + lerna，**多应用 + 共享包**并存；各应用包内 `src/` 结构参考单项目。

## 根目录与 packages 划分

```
根目录/
├── packages/
│   ├── app-admin/          # 管理端应用
│   │   └── src/            # 结构同单项目（api、components、views、stores 等）
│   ├── app-web/            # 前台应用（可多个 app-*）
│   ├── shared-ui/          # 共享 UI 组件
│   ├── shared-utils/       # 共享工具函数
│   └── shared-types/       # 共享类型与接口
├── pnpm-workspace.yaml
├── lerna.json
└── package.json
```

- **应用包**（`app-*`）：每个包内 `src/` 与单项目一致（api、components、composables、layouts、router、stores、styles、types、utils、views、App.vue、main.ts）。
- **共享包**（`shared-*`）：按包职责组织（如 shared-ui 下 components、styles；shared-utils 下按功能划分子文件）。
- **构建顺序**：先构建 shared-*，再构建 app-*；根脚本通过 lerna/pnpm 编排。

通用约定（文件行数、大小、tests/examples）见 [shared/02-目录规范.md](../shared/02-目录规范.md)。
