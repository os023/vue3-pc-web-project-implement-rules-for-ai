# B-08 异步与错误（联邦项目）

联邦项目下**各应用**独立配置请求与拦截器，公共错误与业务错误边界在**本 app 内**统一。

- **写法**：异步逻辑统一使用 `try/catch`，并给出明确兜底（默认值、提示、降级行为）。
- **公共错误**：在**各 app** 的 HTTP/请求拦截器中统一处理（如 401 跳转登录、5xx 提示、网络错误提示）；可在 shared-utils 中提供统一错误码或错误处理工具，供各 app 的拦截器调用。
- **业务错误**：在对应页面或 composable 中就近处理（如接口返回 code !== 200 时的提示或分支逻辑）。

与 A/C 共用规则一致，见 [shared/08-异步与错误.md](../shared/08-异步与错误.md)。
