# B-04 页面规范（联邦项目）

联邦项目下**每个应用**独立拥有自己的页面树与路由，无跨应用共用「页面」。

- **组织**：各 app 内与单项目一致——一个页面一个目录，目录名与路由 path 对应；入口文件统一为 `index.vue`（或与路由 name 一致）。
- **示例**：`packages/app-admin/src/views/user/list/index.vue` 对应该 app 的路由 `/user/list`。
- **职责**：页面负责本应用内的数据拉取、状态与用户操作；复杂逻辑抽到本 app 的 composables 或 shared-utils。
- **路由**：各 app 独立配置在自身 `src/router/`，与**该 app** 的 `views/` 目录结构对应；无跨应用路由。

### 示例（Vue3 页面：script setup + try/catch 兜底）

```vue
<!-- packages/app-admin/src/views/user/list/index.vue -->
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { getUserList } from '@/api/user' // 本 app 内 api

const list = ref<{ id: number; name: string }[]>([])
const loading = ref(false)

onMounted(async () => {
  loading.value = true
  try {
    const res = await getUserList()
    list.value = res?.data ?? []
  } catch {
    list.value = []
  } finally {
    loading.value = false
  }
})
</script>

<template>
  <div class="os023-page os023-user-list">
    <div v-if="loading">加载中...</div>
    <ul v-else>
      <li v-for="item in list" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>
```

共用规则（组织方式、职责、路由约定）见 [shared/04-页面规范.md](../shared/04-页面规范.md)。
