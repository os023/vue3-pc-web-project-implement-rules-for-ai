# A-02 目录规范（单项目）

单项目下仅一个应用，推荐采用以下**单一 `src` 目录**结构。

## 单项目推荐结构

```
src/
├── api/              # 接口封装，按业务或模块划分子文件
├── assets/           # 静态资源（图片、字体等）
├── components/       # 公共组件（见 A-03）
│   └── base/         # 可选：基础 UI 组件
├── composables/      # 组合式函数
├── layouts/          # 布局组件
├── router/           # 路由配置
├── stores/           # Pinia 状态（见 A-05）
├── styles/           # 全局样式、变量、主题
├── types/            # 全局类型、接口定义
├── utils/            # 工具函数
├── views/            # 页面级组件（见 A-04）
├── App.vue
└── main.ts
```

- 入口唯一：`main.ts` + `App.vue`，一套环境变量与构建配置。
- 无 `packages/`：所有业务与公共代码均在本 `src/` 下，通过目录划分职责。
