# A-03 公共组件规范（单项目）

单项目下公共组件与业务组件均在同一仓库内，通过**目录位置**区分。

- **位置**
  - 公共组件：`src/components/`；可选子目录 `base/` 放基础 UI 组件。
  - 与业务强绑定的组件：放在对应页面目录下，如 `views/user/list/components/`。
- 命名、Props/Emits、边界、文档见 [shared/03-公共组件规范.md](../shared/03-公共组件规范.md)。

### 示例（Vue3 script setup + 类型）

```vue
<!-- src/components/UserAvatar.vue -->
<!-- 用于展示用户头像，支持尺寸与占位图 -->
<script setup lang="ts">
interface Props {
  src?: string
  size?: 'sm' | 'md' | 'lg'
}
const props = withDefaults(defineProps<Props>(), { size: 'md' })
const emit = defineEmits<{ (e: 'click'): void }>()
</script>

<template>
  <img
    :src="props.src || '/default-avatar.png'"
    :class="['os023-avatar', `os023-avatar--${props.size}`]"
    @click="emit('click')"
  />
</template>
```

其余见 [shared/03-公共组件规范.md](../shared/03-公共组件规范.md)。
