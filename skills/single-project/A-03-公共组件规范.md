# A-03 公共组件规范（单项目）

单项目下公共组件与业务组件均在同一仓库内，通过**目录位置**区分。

- **位置**
  - 公共组件：`src/components/`；可选子目录 `base/` 放基础 UI 组件。
  - 与业务强绑定的组件：放在对应页面目录下，如 `views/user/list/components/`。
- **命名**：PascalCase，多词（如 `UserAvatar`、`DataTable`）。
- **Props/Emits**：用 `defineProps`/`defineEmits` 且带类型；复杂 Props 用接口定义在组件文件内或 `src/types/`。
- **边界**：放在 `src/components/` 的组件不直接依赖业务 API 或业务 store；通过 props/events 与父级通信。
- **文档**：组件顶部用中文注释说明用途、主要 props。

### 示例（Vue3 script setup + 类型）

```vue
<!-- src/components/UserAvatar.vue -->
<!-- 用于展示用户头像，支持尺寸与占位图 -->
<script setup lang="ts">
interface Props {
  src?: string
  size?: 'sm' | 'md' | 'lg'
}
const props = withDefaults(defineProps<Props>(), { size: 'md' })
const emit = defineEmits<{ (e: 'click'): void }>()
</script>

<template>
  <img
    :src="props.src || '/default-avatar.png'"
    :class="['os023-avatar', `os023-avatar--${props.size}`]"
    @click="emit('click')"
  />
</template>
```
