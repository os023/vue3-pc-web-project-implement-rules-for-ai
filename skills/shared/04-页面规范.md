# 页面规范（共用）

- **组织**：一个页面一个目录，目录名与路由 path 对应；入口文件统一为 `index.vue`（或与路由 name 一致）。
- **示例**：`views/user/list/index.vue` 对应路由 `/user/list`。
- **职责**：页面负责数据拉取、状态与用户操作；复杂逻辑抽到 `composables` 或 `utils`。
- **路由**：集中配置在 `router/`，与 `views` 目录结构对应，便于维护。

## 页面目录结构

目录层级与路由 path 对应，每页一个目录，入口统一为 `index.vue`；仅本页使用的子组件与组合式函数可放在该页目录下：

```
...views/
└── user/                    # 对应路由 /user/...
    └── list/                # 对应路由 /user/list
        ├── index.vue        # 页面入口，对外即「该页」
        ├── components/      # 仅本页使用的子组件（可选）
        │   ├── ListTable.vue
        │   └── ListFilter.vue
        ├── hooks/           # 仅本页使用的组合式函数（可选）
        │   └── useListData.ts
        └── utils/           # 仅本页使用的工具函数（可选）
            └── formatUtil.ts
```

- 简单页面可仅保留 `list/index.vue`，无需 `components/`、`hooks/` 等子目录。
- 单项目/联邦项目/联邦库下的差异（如多应用路由、库无页面等）见各模式 04-页面规范。
