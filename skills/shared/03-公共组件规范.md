# 公共组件规范（共用）

- **命名**：PascalCase，多词（如 `UserAvatar`、`DataTable`）。
- **Props/Emits**：用 `defineProps`/`defineEmits` 且带类型；复杂 Props 用接口定义在组件文件内或类型目录。
- **边界**：公共组件不直接依赖业务 API 或业务 store；通过 props/events 与父级通信。
- **文档**：组件顶部用中文注释说明用途、主要 props。

## 组件目录结构

单文件组件可直接放在 `components/` 下；多文件、带子逻辑的组件建议按「一组件一目录」组织，目录名与组件名一致（PascalCase）：

```
...components/
└── UserAvatar/           # 组件名即目录名
    ├── index.ts         # 统一导出，便于外部 import
    └── src/
        ├── UserAvatar.vue   # 主组件
        ├── common/          # 本组件内共用的类型、常量、子组件
        └── hooks/           # 本组件内共用的组合式函数
```

- 简单组件可仅保留 `UserAvatar.vue` 单文件，无需目录。
