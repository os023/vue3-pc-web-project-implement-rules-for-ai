# C-08 异步与错误（联邦库）

联邦库内异步逻辑**统一 try/catch、明确兜底**；库不负责 UI 层面的「公共错误拦截」，由使用方（主应用）在拦截器或页面中处理。

- **写法**：异步逻辑统一使用 `try/catch`，对外 API 明确错误形态（抛错或返回 Result）；不在库内做全局错误弹窗或跳转。
- **文档**：库的 README 或类型定义中说明可能抛出的错误或错误码，便于主应用在拦截器或页面中处理。
- **主应用**：公共错误仍在主应用的 HTTP/请求拦截器中处理；业务错误在页面中就近处理；若库封装了请求，主应用需能拿到错误并统一处理。

与 A/B 共用规则一致，见 [shared/08-异步与错误.md](../shared/08-异步与错误.md)。
