# C-01 推荐使用的 npm 库（联邦库）

联邦库（pnpm + lerna）下，各**库包**以发布产物为主；推荐在库内统一使用以下库，便于与使用方（主应用）一致、减少重复打包。

| 用途 | 推荐库 | 说明 |
|------|--------|------|
| 包管理 / workspace | pnpm | 根与各库包统一使用 pnpm；配合 pnpm-workspace.yaml |
| 多包编排 / 发布 | lerna | 多包版本与发布编排，与 pnpm 配合使用 |
| 时间处理 | dayjs | 库内涉及日期均用 dayjs。详见 [C-06-时间处理.md](C-06-时间处理.md) |
| 数组/对象/判断 | lodash-es | 按需引入，控制体积。详见 [C-07-数组对象判断.md](C-07-数组对象判断.md) |
| 状态管理 | 不内置 | 库不内置 Pinia；由使用方管理状态。详见 [C-05-状态管理.md](C-05-状态管理.md) |
| 路由 | 不内置 | 由使用方配置 |
| HTTP | 按需 | 若库封装接口，可依赖 axios 等；敏感配置由使用方注入 |
| 图标 | @iconify/vue 或 Iconify 生态 | 与 [C-10-样式规范](C-10-样式规范.md) 一致 |
| 工具/校验等 | 按需选型 | 优先 lodash-es；对外 API 不直接 re-export 整库 |

- 库包将 dayjs、lodash-es 等声明为 dependency 或 peerDependency，按发布策略选择；产物为 ESM（及可选 CJS），供主应用按需引入。
- 新增通用能力时优先在以上范围内选型；若引入新库，需在库 README 或本约定中补充说明。

共用规则见 [shared/01-推荐使用的npm库.md](../shared/01-推荐使用的npm库.md)。
