# C-09 类型与代码质量（联邦库）

联邦库下**每个库包**均发布类型（.d.ts 或 package.json 的 types 字段）；TypeScript 严格模式；构建顺序为先 types/utils，再依赖它们的 ui。

- **TypeScript**：全仓库严格模式；对外导出的函数、组件、接口均定义类型。
- **类型包**：若存在独立 `packages/types`，被其他库包与主应用引用；组件库、工具库自身也可包含类型定义，通过 package.json 的 types/exports 暴露。
- **代码质量**：ESLint + Prettier；遵循 Vue3 与 TypeScript 最佳实践；注意工具函数复用。版本与变更建议用 changesets 或 lerna version 管理。

共用规则见 [shared/09-类型与代码质量.md](../shared/09-类型与代码质量.md)。
