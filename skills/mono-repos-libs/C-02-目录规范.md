# C-02 目录规范（联邦库）

联邦库下根目录为 pnpm workspace + lerna，**多个库包**并存；每个库包具备清晰入口（main、module、types），无「单应用 src」结构。

## 根目录与 packages 划分

```
根目录/
├── packages/
│   ├── ui/                 # Vue 组件库
│   │   ├── src/
│   │   │   ├── components/
│   │   │   ├── styles/
│   │   │   └── index.ts    # 统一导出
│   │   ├── package.json    # main、module、exports
│   │   └── vite.config.ts  # 或 rollup 等
│   ├── utils/              # 纯 TS/JS 工具库
│   │   ├── src/
│   │   │   └── index.ts
│   │   └── package.json
│   └── types/              # 类型包
│       ├── src/
│       └── package.json
├── examples/               # 库的使用示例（可选，也可在各包内）
├── pnpm-workspace.yaml
├── lerna.json
└── package.json
```

- **库包**：每个包有独立 `src/` 与构建产物；组件库按组件名建子目录，含组件与样式；工具库、类型包按功能划分子文件。
- **入口**：每个库包在 package.json 中配置 main、module、types（或 exports），支持按需引入。
- **构建顺序**：先构建 types、utils 等被依赖包，再构建依赖它们的 ui 等。
